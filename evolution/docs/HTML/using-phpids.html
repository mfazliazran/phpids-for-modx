Using PHPIDS

To see possible attacks, that are logged to the database, click on "Modules" ⇒ "PHPIDS". This does only work, if you have configured the module with Use database? = True.

There are two tab-sheets, one shows the list of possible attacks, and the other one shows a list of blocked IP addresses.

The table are both completely sortable by clicking on the header of a column.

The Log Data tab

The Log Data tab shows you possible impacts:

The IP address of the attacker is a link, that shows further information in the RIPE database.

If you want to block an attacker by his IP address, you have to click on the button "Block IP". After you've confirmed, that you want to block the shown IP Address, all users with this IP address are forwarded as defined in Redirect intrusion to the document ID (0 means no redirection). For example use a static HTML page, where an attacker can do no harm to your server.

The Blocked Data tab

On the Blocked Data tab you see the currently blocked IP addresses, that are redirected.

The IP address of the attacker is a link, that shows further information in the RIPE database.

The Extended Options tab

This checks, whether there is a filter update available at phpids.org, or not. Currently there are three possible messages:

  • There is no filter update available.
  • There is a filter update from {update file date} available
  • PHPIDS Filter URI is not available.

No filter update available means, that everything is OK. If there is a filter update available, then there are two possible options:

If you have full rights on the filter file, located in the root directory of your PHPIDS lib folder, then you only have to push the button "Update Filter". Than the new filter data will be read from phpids.org and written to your filter.

The second option is to download the filter from phpids.org and upload it manually to your system.

"PHPIDS Filter URI is not available" means, that the module can't connect to the filter. This may be for example, that there is a problem at phpids.org.

Download the intrusion data as CSV file

With the link "Download intrusions as CSC" file, you get the complete data of the intrusion table. The delimiter and the enclosure for the CSV files are configurable at the module configuration.

Empty log table

It is now possible, to empty the log table, but afterward all entries about previous possible attacks are lost.
